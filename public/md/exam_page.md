# 응시자 자료실 (Exam Materials) 개발 계획

## 1. 개요

* **목표**: '응시자 자료실'을 기존 게시판 형태에서 **카드형 그리드 레이아웃**으로 개편하고, 관리자 전용 관리 페이지를 구축합니다.
* **대상 보드**: `EXAM_USER` (응시자 자료실)
* **주요 기능**:
  * 관리자: 자료 등록/수정/삭제 (PDF, TXT), 그리드 설정.
  * 사용자: 카드형 목록 조회, 자료 열람 (클릭 시 실행).

## 2. 상세 요구사항 분석

### A. 관리자 페이지 (Admin)

* **메뉴 위치**: 사이드바 **최하위**에 '응시자 관리' 메뉴 추가.
* **기능**:
  * **자료 목록 관리**: `EXAM_USER` 게시글 CRUD.
  * **자료 등록/수정**:
    * 제목 (카드 제목)
    * 파일 첨부 (PDF, TXT 제한)
  * **설정**:
    * 그리드 열 개수 설정 (예: 3열, 4열). `BoardSettings`에 저장.

### B. 공개 페이지 (Public)

* **경로**: `/board/exam` (기존 페이지 수정)
* **레이아웃**:
  * 탭 구조 유지 (고시부 자료실 / **응시자 자료실**)
  * '응시자 자료실' 탭 선택 시 **그리드 컴포넌트** 렌더링.
* **그리드 카드 디자인**:
  * **배경색**: `#338a37` (짙은 녹색 계열)
  * **제목**: Bold, `#c8cfc9` (연한 회색), 카드 비율에 맞는 큰 폰트.
  * **공통 문구**: 제목 하단에 "남경기노회 고시부" 표시 (작은 폰트, `#d7ddf7`).
  * 형태: 정사각형, 반응형 그리드.
* **동작 (실행 방식)**:
  * **1차 구현**: 카드 클릭 시 첨부된 파일(PDF/TXT)을 **브라우저 새 탭**에서 열기.
  * **최종 목표 (Feasibility Check)**: 카드 클릭 시 **모달**이 뜨고, PDF 파일을 **책장 넘기듯(Page Flip)** 볼 수 있는 뷰어 구현.
* **기타**:
  * 검색 기능 (제목 검색).
  * 글쓰기 버튼 숨김.

## 3. 개발 단계 (Step-by-Step)

### Step 1: 관리자 페이지 구현 (`/admin/exam`)

1. **API 구현**:
    * `GET /api/admin/exam`: 자료 목록 조회.
    * `POST /api/admin/exam`: 자료 등록 (파일 업로드 포함).
    * `PUT/DELETE`: 수정/삭제.
    * `POST /api/board-settings/EXAM_USER`: 그리드 설정 저장.
2. **UI 구현**:
    * 자료 목록 테이블.
    * 등록/수정 모달 또는 페이지.
    * 설정 패널 (그리드 수 조절).

### Step 2: 공개 페이지 컴포넌트 개발

1. **`ExamMaterialList` 컴포넌트 생성**:
    * `PostList` 대신 사용할 전용 컴포넌트.
    * `EXAM_USER` 데이터 Fetching.
    * Grid Layout (Tailwind CSS 활용).
    * **디자인 적용**: `#338a37` 배경, `#c8cfc9` 제목, `#d7ddf7` 서브텍스트.
2. **`/board/exam/page.tsx` 수정**:
    * `activeTab === 'EXAM_USER'`일 때 `ExamMaterialList` 렌더링.

### Step 3: 파일 처리 및 실행 로직

1. **파일 업로드 제한**: PDF, TXT만 허용하도록 `FileUploader` 설정.
2. **카드 클릭 이벤트**:
    * 1차: `window.open()`으로 새 탭 열기.
    * 2차(옵션): PDF Flip Viewer 모달 구현 (라이브러리 검토 후 결정).

## 4. 질문 사항 (To User)

* (완료) 실행 방식: 1차 새 탭, 최종 목표는 Page Flip 효과 모달.
